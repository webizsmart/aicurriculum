# /plan-ceo-review: Strategy & Ambition Review

Load identity: [persona-gstack-plan-ceo-review.md](.antigravity/rules/persona-gstack-plan-ceo-review.md)

## Phase 1: Pre-Review System Audit
1. Run history scan, diff stat, and pain point mapping.
2. Detect Design Doc and Handoff notes. If missing, offer `/office-hours`.
3. Perform Taste Calibration: identify high-quality vs. anti-patterns in existing code.

## Phase 2: Nuclear Scope Challenge (Step 0)
1. **Premise Challenge:** Is this the right problem? business outcome? direct path?
2. **Landscape Check:** Research category landscape, alternatives, and Layer 3 (first principles).
3. **Dream State Mapping:** 12-month ideal envisioning.
4. **Implementation Alternatives:** Present at least 2 distinct approaches (Minimal vs. Ideal).

## Phase 3: Mode Selection & Expansion
1. User selects Mode: **EXPANSION** (dream big), **SELECTIVE EXPANSION** (cherry-pick), **HOLD SCOPE** (rigor), or **SCOPE REDUCTION** (surgeon).
2. For Expansion modes: run the 10x check, platonic ideal, and delight opportunity scan.
3. Execute the Expansion Opt-in / Cherry-pick ceremony.
4. Persist the CEO Plan and run the adversarial Spec Review Loop (reviewer subagent).

## Phase 4: Execution Review
1. **Architecture Review:** Dependency graphs, ASCII diagrams for data flows/state machines.
2. **Error & Rescue Map:** trace every failure mode and exception class (Zero Silent Failures).
3. **Security & Threat Model:** Attack surface, input validation, auth scoping.
4. **Data Flow & Interaction Edge Cases:** Trace happy/nil/empty/error paths.

## Phase 5: Reporting & Handoff
1. Produce THE REQUIRED OUTPUTS: "NOT in scope", "What already exists", `TODOS.md` updates.
2. Display the Completion Summary and update the Review Readiness Dashboard in the plan file.
